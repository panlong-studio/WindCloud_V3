## 6. 联调中发现并修复的问题

本次联调不是一次通过，中间发现了两个真实问题，并已修复。

### 问题 1：下载时找不到真实文件

现象：

- 数据库里 `HEX(sha256sum)` 返回的是大写字符串
- 真实磁盘文件名是客户端上报的 hash，小写
- Linux 文件名区分大小写
- 所以下载时会把路径拼成大写 hash，导致 `open()` 失败

修复：

- 在 [src/server/dao_file.c](/home/liwenshuo/my_project/WindCloud_V3/src/server/dao_file.c) 中，把

```sql
SELECT HEX(sha256sum)
```

改成

```sql
SELECT LOWER(HEX(sha256sum))
```

### 问题 2：同路径重复上传时客户端卡住

现象：

- 服务端发现重名路径后，直接回了普通文本包
- 但客户端此时还在等 `file_packet_t`
- 导致协议错位，客户端阻塞

修复：

- 在 [src/server/file_transfer.c](/home/liwenshuo/my_project/WindCloud_V3/src/server/file_transfer.c) 中
- 同路径重复上传时，先回一个 `file_packet_t`
- 再补一条普通文本 `"错误：该文件已存在"`

这样客户端先把协议走完整，再收到错误提示，不会卡住
