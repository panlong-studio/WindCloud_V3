## 数据库说明：netdisk_db

`netdisk_db` 是一个用于实现**网络云盘系统**的底层数据库。其核心设计理念是**“逻辑目录与物理文件分离”**，通过这种设计可以轻松实现文件“秒传”和“去重”功能。该数据库共包含三张核心数据表：`users`（用户表）、`files`（物理文件表）和 `paths`（逻辑路径表）。

------

### 1. 表：users（用户信息表）

该表用于存储云盘系统中注册用户的基本账号信息和安全认证数据。

- **设计细节与说明：**
  - **`id`**: 用户的唯一标识（主键），自增。
  - **`username`**: 用户名，设置了 `UNIQUE`（唯一）约束，确保用户名不重复。
  - **`password_hash`**: 存储用户密码的哈希值（固定长度 64 位字符），不存储明文密码，保障数据安全。
  - **`salt`**: 密码加盐值（固定长度 32 位字符），与原密码结合后进行哈希运算，用于抵御彩虹表攻击。
  - **存储引擎**: InnoDB。

------

### 2. 表：files（物理文件表）

该表用于记录实际上传到云盘服务器的**物理文件信息**。网盘系统基于此表实现文件的“去重”存储。不论有多少个用户上传了同一个文件，物理存储上只保留一份。

- **设计细节与说明：**
  - **`id`**: 物理文件的唯一标识（主键），自增。
  - **`sha256sum`**: 文件的 SHA-256 校验和（使用 `BINARY(32)` 存储，节省空间且比字符串查找更快）。该字段为 `UNIQUE`（唯一），系统通过它判断文件是否已被上传过，从而实现“秒传”。
  - **`size`**: 物理文件的实际大小（字节）。
  - **`count`**: 引用计数器（默认值为 0）。用于记录该物理文件被多少个虚拟路径（用户网盘文件）引用。当用户删除一个文件时，只需将该计数器减 1；只有当计数器归 0 时，才真正从服务器磁盘上删除物理文件。
  - **存储引擎**: InnoDB。

------

### 3. 表：paths（逻辑路径表 / 虚拟文件系统）

该表是网盘系统的核心业务表，记录了每个用户在自己网盘中看到的**虚拟文件和目录树形结构**。它将用户可见的文件名与底层的物理文件关联起来。

- **核心逻辑说明：**
  - **关于 `type` 与 `file_id` 的联动：**
    - 当 **`type = 0`**（普通文件）时：表示这是一个具体的文件，此时 **`file_id` 存储的是 `files` 表中的 `id`**，指向实际的物理文件。
    - 当 **`type = 1`**（目录/文件夹）时：表示这是一个文件夹，此时没有对应的物理文件实体，因此 **`file_id` 为 `NULL`（空）**。
- **设计细节与说明：**
  - **`id`**: 路径记录的唯一标识（主键），自增。
  - **`user_id`**: 归属用户的 ID，关联 `users` 表。
  - **`path`**: 文件的完整虚拟路径记录。
  - **`parent_id`**: 父目录的 `id`，用于构建目录的层级/树状结构（例如根目录的 parent_id 可以约定为 0）。
  - **`file_name`**: 用户可见的文件名或文件夹名。
  - **索引与性能优化：**
    - **`UNIQUE KEY (user_id, path)`**: 复合唯一键。确保同一个用户在自己的网盘内，不可能存在两个完全相同的完整路径（防止同一目录下出现同名文件/文件夹）。
    - **`INDEX idx_user_parent (user_id, parent_id)`**: 复合普通索引。在实际业务中，“查询某个用户在某个文件夹下的所有内容”是非常高频的操作。该索引能大幅提升展示特定目录下文件列表时的查询速度。
  - **存储引擎**: InnoDB。
